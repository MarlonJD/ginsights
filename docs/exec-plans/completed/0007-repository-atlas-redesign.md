# 0007 Repository Atlas Redesign

## Status

Complete.

## Goal

Redesign the static `ginsights` report from the ground up so it feels like a polished local repository insight product, not an engineering debug dashboard.

## Surface, Role, Job

- Surface: self-contained HTML report generated by `internal/report`.
- User role: developer or maintainer inspecting a local/private repository.
- Single job: understand repository activity, ownership, churn, language mix, and maintenance signals from offline data.

## Product Shape

Classify the report as an investigation surface with a dossier-like first view. It should help a maintainer scan evidence and decide where to look next, without implying productivity ranking or requiring network data.

## Signature Element

Repository Atlas: a wide first viewport that treats local Git history as a navigable atlas. The hero should combine repo identity, a 53-week activity strip, net-change balance, and compact evidence markers for health, primary language, and provenance.

## Visual Direction

- Use one disciplined signature element: the atlas hero/activity strip.
- Keep everything else operational, dense, and readable.
- Avoid remote assets, frontend build tooling, raw decorative flourishes, and fake metrics.
- Preserve progressive enhancement: HTML remains readable without JavaScript.

## Critique Before Coding

Generic choices to avoid:

- SaaS metric card grid as the first impression.
- Decorative gradients unrelated to repo evidence.
- Oversized marketing hero copy.
- Tables that look like dumped debug output.
- A sidebar that makes the page feel like an admin shell.

Revision from critique:

- Replace the sidebar-admin composition with a compact top rail and anchored sections.
- Put the commit activity visualization in the first viewport as the report's signature.
- Convert secondary data into editorial but scannable evidence panels.
- Keep tables for accessibility, but make them feel like drill-down evidence rather than the primary design.

## Scope

- `internal/report/report.go` HTML/CSS/view helpers as needed.
- `internal/report/report_test.go` and golden fixture updates.
- Generated screenshot asset if visual verification produces a new current capture.

## Non-goals

- Do not add a frontend build step, JavaScript chart dependency, remote asset, or UI framework.
- Do not clone GitHub Insights branding.
- Do not add or imply GitHub Traffic metrics in offline core mode.
- Do not rank contributors by productivity.

## Acceptance criteria

- The first viewport uses a purpose-built Repository Atlas layout rather than an admin/sidebar dashboard.
- The commit activity area uses available horizontal space without leaving large inert gaps.
- Recent commits and code frequency do not visually overlap at desktop or mobile widths.
- The report remains self-contained static HTML and readable without JavaScript.
- Renderer tests and the golden fixture reflect the intentional markup changes.
