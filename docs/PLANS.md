# Plans

Plans live in `docs/exec-plans/`.

Use active plans for multi-step work. Move a plan to `completed/` when done and leave a short completion note.

Required active-plan headings:

- Status
- Goal
- Scope
- Non-goals
- Acceptance criteria
- Verification
- Decision log
- Next actions

Small one-file fixes can skip a formal plan, but the final response still needs checks run and any assumptions named.
