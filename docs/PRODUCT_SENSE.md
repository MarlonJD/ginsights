# Product sense

## Positioning

`ginsights` is not “GitHub Insights with forbidden data.” It is “GitHub-style repository insights from data you already have locally.”

## Differentiators

- single Go binary;
- no upload;
- no required token;
- static export;
- Codex-friendly architecture and plans;
- mechanical repo-harness checks.

## Tone

Avoid productivity-score framing. Prefer repository maintenance language:

- activity;
- churn;
- hotspots;
- review/quality signals;
- health.
